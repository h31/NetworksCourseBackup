\chapter{Задание}
Задание: разработать приложение–клиент и приложение–сервер
электронного магазина. Товар в электронном магазине имеет уникальный
идентификатор, именование, цену. 
\section{Функциональные требования}
Серверное приложение должно реализовывать следующие функции:
\begin{enumerate}
\item{Прослушивание определенного порта}
\item{Обработка запросов на подключение по этому порту от клиентов
электронного магазина}
\item{Поддержка одновременной работы нескольких клиентов электрон-
ного магазина через механизм нитей}
\item{Прием запросов на добавление или покупку товара}
\item{Осуществление добавления товара, учет количества единиц товара}
\item{Передача клиенту электронного магазина информации о товарах и
подтверждений о совершении покупки}
\item{Обработка запроса на отключение клиента}
\item{Принудительное отключение клиента}
\end{enumerate}

Клиентское приложение должно реализовывать следующие функции:
\begin{enumerate}
\item{Установление соединения с сервером}
\item{Передача запросов о добавлении, покупке товаров серверу}
\item{Получение ответов на запросы от сервера}
\item{Разрыв соединения}
\item{Обработка ситуации отключения клиента сервером}
\end{enumerate}

\chapter{Реализация для работы по протоколу TCP}
\section{Прикладной протокол}
Клиент отправляет команды в виде строки с текстом, заканчивающейся символом перевода строки (\textbackslash n).
\begin{itemize}
\item show index - показать все доступные в магазине товары
\item show cart - показать содержимое корзины
\item buy N - купить товар под номером N
\item add N P C - добавить в каталог товар с именем N по цене P в количестве C
\item close - закрыть сессию
\end{itemize}
Формат ответа на команды show cart и show index:

\begin{verbatim}
I N P C
\end{verbatim}

Где I - номер товара в магазине, N - его название, P - его цена, C - количество.
